# Sparse Table（稀疏表）

稀疏表是一种用于**静态区间查询**的数据结构，支持在 O(1) 时间内回答区间最值、区间 GCD 等查询，预处理时间复杂度为 O(n log n)。

## 核心思想

将数组拆分为长度为 2 的幂次的子区间，预先计算每个子区间的聚合结果。查询时用两个子区间覆盖目标区间。

## 构建公式

设 `st[i][j]` 表示从下标 `i` 开始、长度为 `2^j` 的区间的聚合结果。

**初始化**（长度为 1 的区间）：

```
st[i][0] = arr[i]
```

**递推**（长度为 2^j 的区间由两个长度为 2^(j-1) 的区间合并）：

```
st[i][j] = cal(st[i][j-1], st[i + 2^(j-1)][j-1])
```

其中 `cal` 为聚合函数（如 max、min、gcd）。

**循环顺序**：外层遍历 `j`（层数），内层遍历 `i`（起始位置），确保计算 `st[i][j]` 时依赖的子区间已被计算。

## 查询公式

查询区间 `[l, r]`，长度 `len = r - l + 1`。

取 `k = floor(log₂(len))`，即不超过 `len` 的最大 2 的幂次。

用两个长度为 `2^k` 的区间覆盖 `[l, r]`：

```
result = cal(st[l][k], st[r - 2^k + 1][k])
```

- `st[l][k]` 覆盖 `[l, l + 2^k - 1]`
- `st[r - 2^k + 1][k]` 覆盖 `[r - 2^k + 1, r]`

两个区间有重叠，但对于 MAX、MIN、GCD 等幂等操作，重叠不影响结果。

### 推导过程

**问题**：稀疏表只存储了长度为 2 的幂次的区间（`st[i][0]` 长度 1，`st[i][1]` 长度 2，`st[i][2]` 长度 4……），但查询区间 `[l, r]` 的长度 `len` 不一定是 2 的幂次，无法直接用一个 `st[l][j]` 得到答案。

**思路**：找一个 `k`，使得可以用**两个**长度为 `2^k` 的区间覆盖整个 `[l, r]`。

**为什么两个区间就够了**：两个长度为 `2^k` 的区间最多覆盖 `2 × 2^k = 2^(k+1)` 个元素。只要 `2^(k+1) ≥ len`，两个区间就能覆盖。而 `k = floor(log₂(len))` 保证了 `2^k ≤ len < 2^(k+1)`，因此 `2^(k+1) > len` 成立。

**两个区间的选取**：
- 第一个区间从 `l` 开始：`[l, l + 2^k - 1]`，对应 `st[l][k]`
- 第二个区间在 `r` 处结束：`[r - 2^k + 1, r]`，对应 `st[r - 2^k + 1][k]`

**验证覆盖完整性**：第二个区间的左端点 `r - 2^k + 1`，第一个区间的右端点 `l + 2^k - 1`。两者之差：

```
(r - 2^k + 1) - (l + 2^k - 1) = (r - l + 2) - 2^(k+1) = (len + 1) - 2^(k+1)
```

由于 `2^(k+1) > len`，所以 `(len + 1) - 2^(k+1) ≤ 0`，即第二个区间的左端点 ≤ 第一个区间的右端点，说明两个区间**有重叠或刚好衔接**，覆盖了 `[l, r]` 的全部元素。

**为什么重叠可以接受**：对于 MAX、MIN、GCD 等幂等操作（`cal(x, x) = x`），同一元素被计算两次不影响结果。

### 为什么 k 向下取整

向上取整会导致 `2^k > len`，区间长度超过查询范围，起始位置加上这个长度会数组越界。

向下取整保证 `2^k ≤ len`，不会越界。

## 复杂度

| 操作     | 时间复杂度  | 空间复杂度  |
| -------- | ----------- | ----------- |
| 预处理   | O(n log n)  | O(n log n)  |
| 单次查询 | O(1)        | -           |

## 局限性

- 仅支持**静态数组**，不支持修改元素
- 仅适用于**幂等操作**（重复计算不影响结果），如 MAX、MIN、GCD
- 非幂等操作（如求和）需要使用其他数据结构（如线段树）
