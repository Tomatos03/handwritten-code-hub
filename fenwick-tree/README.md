# 树状数组（Fenwick Tree）

## 1. 概述

### 1.1 定义

树状数组（Fenwick Tree），也称为二叉索引树（Binary Indexed Tree, BIT），是一种用于高效处理前缀和查询和单点更新的数据结构。

### 1.2 核心特点

- **高效操作**：单点更新和前缀和查询的时间复杂度均为 O(log n)
- **空间紧凑**：仅需 O(n) 的额外空间
- **实现简洁**：代码量少，易于理解和实现
- **常数因子小**：相比线段树，实际运行速度更快

### 1.3 时间复杂度

| 操作 | 时间复杂度 |
|------|-----------|
| 建树（批量） | O(n) |
| 建树（逐个插入） | O(n log n) |
| 单点更新 | O(log n) |
| 前缀和查询 | O(log n) |
| 区间和查询 | O(log n) |

---

## 2. 数据结构原理

### 2.1 lowbit 运算

树状数组的核心是 `lowbit` 运算，它返回一个整数的最低位 1 所代表的值：

```
lowbit(x) = x & (-x)
```

**示例**：
- `lowbit(6)` = `6 & (-6)` = `0110 & 1010` = `0010` = 2
- `lowbit(8)` = `8 & (-8)` = `1000 & 1000` = `1000` = 8
- `lowbit(12)` = `12 & (-12)` = `1100 & 0100` = `0100` = 4

### 2.2 数组存储

树状数组使用一个一维数组 `tree[]` 存储，**下标从 1 开始**（1-indexed），索引 0 不使用。

对于数组 `tree[i]`，它存储的是原数组中某个区间的和：
- `tree[i]` 负责的区间长度为 `lowbit(i)`
- `tree[i]` 负责的区间为 `[i - lowbit(i) + 1, i]`

### 2.3 树形结构

树状数组的逻辑结构是一棵树，每个节点负责管理一定范围的元素：

```
原数组下标:  1   2   3   4   5   6   7   8
           ─┼───┼───┼───┼───┼───┼───┼───→
tree[1]:   [1]                                 负责 [1,1]
tree[2]:   [1 ─── 2]                           负责 [1,2]
tree[3]:            [3]                        负责 [3,3]
tree[4]:   [1 ─── 2 ─── 3 ─── 4]              负责 [1,4]
tree[5]:                        [5]            负责 [5,5]
tree[6]:                        [5 ─── 6]      负责 [5,6]
tree[7]:                                 [7]   负责 [7,7]
tree[8]:   [1 ─── 2 ─── 3 ─── 4 ─── 5 ─── 6 ─── 7 ─── 8]  负责 [1,8]
```

**父子关系**：
- 节点 `i` 的父节点：`i + lowbit(i)`
- 节点 `i` 的子节点：`i - lowbit(i)`

## 3. 核心操作详解

### 3.1 建树（Build）

#### 朴素方法：逐个插入

将原数组的每个元素逐个调用 `add` 方法插入，时间复杂度为 O(n log n)。

#### 优化方法：O(n) 批量建树

利用树状数组的性质，可以在 O(n) 时间内完成建树：

```java
// 1. 先将原数组复制到 tree 中
for (int i = 1; i <= n; i++) {
    tree[i] = nums[i - 1];
}
// 2. 对每个节点，将其值加到其父节点上
for (int i = 1; i <= n; i++) {
    int j = i + lowbit(i);  // 父节点
    if (j <= n) {
        tree[j] += tree[i];
    }
}
```

**图解过程**：

以数组 `[1, 3, 5, 7, 9, 11]` 为例（n=6）：

```
初始状态：tree = [0, 1, 3, 5, 7, 9, 11]

处理 i=1: j = 1 + lowbit(1) = 2, tree[2] += tree[1] → tree[2] = 3+1 = 4
处理 i=2: j = 2 + lowbit(2) = 4, tree[4] += tree[2] → tree[4] = 7+4 = 11
处理 i=3: j = 3 + lowbit(3) = 4, tree[4] += tree[3] → tree[4] = 11+5 = 16
处理 i=4: j = 4 + lowbit(4) = 8, 8 > 6，跳过
处理 i=5: j = 5 + lowbit(5) = 6, tree[6] += tree[5] → tree[6] = 11+9 = 20
处理 i=6: j = 6 + lowbit(6) = 8, 8 > 6，跳过

最终 tree = [0, 1, 4, 5, 16, 9, 20]
```

---

### 3.2 单点更新（Update）

#### 算法思路

在位置 `pos` 加上值 `v` 时，需要更新所有包含 `pos` 的区间，即从 `tree[pos]` 开始，沿着父节点路径向上更新。

#### 实现代码

```java
public void add(int pos, long v) {
    while (pos <= n) {
        tree[pos] += v;
        pos += lowbit(pos);  // 移动到父节点
    }
}
```

#### 图解过程

在位置 3 加上值 2（原数组 `[1, 3, 5, 7, 9, 11]`）：

```
更新路径：3 → 4 → 8

tree[3] += 2  (负责 [3,3])
tree[4] += 2  (负责 [1,4])
tree[8] 超出范围，停止

更新前 tree: [0, 1, 4, 5, 16, 9, 20]
更新后 tree: [0, 1, 4, 7, 18, 9, 20]
```

---

### 3.3 前缀和查询（Query）

#### 算法思路

查询前 `x` 项的和时，需要将 `[1, x]` 区间分解为若干个不相交的子区间，这些子区间的和恰好覆盖 `[1, x]`。

#### 实现代码

```java
public long query(int x) {
    long cnt = 0;
    while (x > 0) {
        cnt += tree[x];
        x -= lowbit(x);  // 移动到负责更小区间的位置
    }
    return cnt;
}
```

#### 图解过程

查询前 5 项的和（`query(5)`）：

```
查询路径：5 → 4 → 0

tree[5] += 9   (负责 [5,5])
tree[4] += 16  (负责 [1,4])
x = 0，停止

结果 = 9 + 16 = 25
```

---

### 3.4 区间和查询（Sum）

#### 算法思路

查询区间 `[l, r]` 的和，利用前缀和的性质：

```
sum(l, r) = query(r) - query(l-1)
```

#### 实现代码

```java
public long sum(int l, int r) {
    return query(r) - query(l - 1);
}
```

#### 示例

查询区间 `[2, 5]` 的和：

```
sum(2, 5) = query(5) - query(1)
         = (tree[5] + tree[4]) - tree[1]
         = (9 + 16) - 1
         = 24
```

## 4. 应用场景

### 4.1 前缀和查询

**典型问题**：动态计算数组的前缀和，支持单点修改。

**应用场景**：
- 实时统计销售额的累计值
- 游戏中的分数累计系统

### 4.2 逆序对计数

**典型问题**：统计数组中逆序对的数量。

**算法思路**：
1. 从右向左遍历数组
2. 对于每个元素，查询已插入元素中小于它的个数
3. 将当前元素插入树状数组

**时间复杂度**：O(n log n)

### 4.3 动态区间统计

**典型问题**：
- 区间不同元素个数
- 区间第 K 大元素
- 二维树状数组（矩阵区域和）

### 4.4 典型例题

1. **单点更新 + 区间查询**
   - LeetCode 307: Range Sum Query - Mutable

2. **逆序对计数**
   - LeetCode 315: Count of Smaller Numbers After Self

3. **区间更新 + 单点查询（差分树状数组）**
   - 利用差分数组，将区间更新转化为两次单点更新

## 5. 与其他数据结构对比

### 5.1 树状数组 vs 线段树

| 特性 | 树状数组 | 线段树 |
|------|----------|--------|
| **功能** | 主要支持前缀和查询、单点更新 | 支持区间更新、区间查询、多种聚合操作 |
| **实现复杂度** | 简单（约 30 行代码） | 较复杂（约 100+ 行代码） |
| **空间复杂度** | O(n) | O(4n) |
| **常数因子** | 小 | 较大 |
| **通用性** | 较弱（主要支持求和） | 强（支持最值、GCD 等） |
| **区间更新** | 需要差分技巧 | 原生支持（懒标记） |

### 5.2 适用场景分析

**选择树状数组的情况**：
- 只需要单点更新、前缀和查询
- 对常数因子敏感，追求极致性能
- 代码实现需要简洁
- 空间限制严格

**选择线段树的情况**：
- 需要区间更新功能
- 需要支持多种聚合操作（最值、GCD 等）
- 区间查询的区间长度不固定

### 5.3 复杂度对比

| 操作 | 树状数组 | 线段树 |
|------|----------|--------|
| 建树 | O(n) | O(n) |
| 单点更新 | O(log n) | O(log n) |
| 前缀/区间查询 | O(log n) | O(log n) |
| 区间更新 | O(log n)（差分） | O(log n)（懒标记） |

## 6. 实现细节与优化

### 6.1 下标约定

树状数组通常使用 **1-indexed**（下标从 1 开始），原因：
- `lowbit(0) = 0`，会导致死循环
- 父子关系公式更简洁

如果原数组是 0-indexed，需要在使用时进行下标转换：`pos + 1`。

### 6.2 数据类型选择

本实现使用 `long` 类型存储，优点：
- 支持大数值求和，避免溢出
- 适用范围更广

如果数值范围较小，可以使用 `int` 类型节省空间。

### 6.3 空间优化

#### 压缩树状数组

对于稀疏数组（大部分元素为 0），可以使用压缩树状数组，只存储非零元素。

#### 二维树状数组

扩展到二维空间，用于处理矩阵区域和查询：

```java
class BIT2D {
    int[][] tree;
    int n, m;
    
    void add(int x, int y, int val) {
        for (int i = x; i <= n; i += lowbit(i))
            for (int j = y; j <= m; j += lowbit(j))
                tree[i][j] += val;
    }
    
    int query(int x, int y) {
        int sum = 0;
        for (int i = x; i > 0; i -= lowbit(i))
            for (int j = y; j > 0; j -= lowbit(j))
                sum += tree[i][j];
        return sum;
    }
}
```

### 6.4 区间更新技巧

利用差分数组，可以将区间更新转化为两次单点更新：

```java
// 区间 [l, r] 加 val
void rangeAdd(int l, int r, int val) {
    add(l, val);
    add(r + 1, -val);
}

// 查询位置 i 的值
int pointQuery(int i) {
    return query(i);
}
```
