# 二叉搜索树（Binary Search Tree）

## 1. 概述

### 1.1 定义

二叉搜索树（BST）是一种二叉树数据结构，对于树中的每个节点，其左子树所有节点的值均小于该节点的值，右子树所有节点的值均大于该节点的值。左右子树本身也是一棵二叉搜索树。

### 1.2 核心特点

- **有序性**：中序遍历的结果是一个严格递增的有序序列
- **高效查找**：利用二分思想，平均 O(log n) 时间完成查找
- **动态增删**：支持高效的插入和删除操作
- **递归结构**：左右子树本身也是二叉搜索树

### 1.3 时间复杂度

| 操作 | 平均 | 最差（退化为链表） |
|------|------|-------------------|
| 查找 | O(log n) | O(n) |
| 插入 | O(log n) | O(n) |
| 删除 | O(log n) | O(n) |
| 遍历 | O(n) | O(n) |

## 2. 数据结构原理

### 2.1 树形结构

BST 满足：对于任意节点，左子树所有值 < 该节点值 < 右子树所有值。

```
          8
        /   \
       3     10
      / \      \
     1   6      14
        / \    /
       4   7  13
```

- 节点 3 的左子树：{1}，右子树：{6, 4, 7}，均满足 BST 性质
- 中序遍历：1, 3, 4, 6, 7, 8, 10, 13, 14（有序）

### 2.2 节点表示

每个节点存储以下信息：

- **val**：节点的值
- **count**：该值的出现次数（处理重复值）
- **left**：左子节点引用
- **right**：右子节点引用

使用 `count` 字段可以在一个节点中记录重复值，避免存储多个相同值的节点。

### 2.3 BST 有序性验证

验证一棵二叉树是否为 BST，需要确保每个节点的值在允许的范围内：

```
          8  (范围: -∞ ~ +∞)
        /   \
       3     10  (左: -∞ ~ 8, 右: 8 ~ +∞)
      / \      \
     1   6      14  (1: -∞ ~ 3, 6: 3 ~ 8, 14: 10 ~ +∞)
```

递归传递上下界，若节点值超出范围则不是 BST。

## 3. 核心操作详解

### 3.1 查找（Search）

#### 算法思路

从根节点开始，若目标值等于当前节点值则找到；若小于则进入左子树；若大于则进入右子树。

#### 图解过程

查找值 6：

```
          8        6 < 8，进入左子树
        /   \
    →  3     10    6 > 3，进入右子树
      / \      \
    1   6      14  6 == 6，找到！
        / \    /
       4   7  13
```

### 3.2 插入（Insert）

#### 算法思路

从根节点开始，沿 BST 性质找到合适的空位，创建新节点。若值已存在，则增加 count 字段。

#### 图解过程

插入值 5：

```
插入前：              插入后：
      8                    8
    /   \                /   \
   3     10             3     10
  / \      \           / \      \
 1   6      14       1   6      14
    / \    /             / \    /
   4   7  13           4   7  13
                          /
                         5   <- 新节点
```

插入 5：从 8→3→6→4→5，找到 4 的右子节点为空，插入。

### 3.3 删除（Delete）

#### 算法思路

删除分三种情况：

1. **叶子节点**：直接删除
2. **只有一个子节点**：用子节点替代
3. **有两个子节点**：用后继节点（右子树最小值）替代，再删除后继节点

#### 图解过程

删除有两个子节点的节点 3：

```
步骤 1：找到后继节点（右子树最小值 4）

      8                8
    /   \            /   \
  [3]    10   →    [4]    10
  / \      \       / \      \
 1   6      14   1   6      14
    / \    /         / \    /
   4   7  13        5   7  13
    ↑
  后继节点

步骤 2：用后继节点 4 替代 3，删除原 4 节点

      8
    /   \
   4     10
  / \      \
 1   6      14
    / \    /
   5   7  13
```

### 3.4 遍历

#### 中序遍历（In-Order）

左 → 根 → 右，结果为有序序列。

```
      8
    /   \
   3     10
  / \      \
 1   6      14

中序遍历：1 → 3 → 6 → 8 → 10 → 14（有序）
```

#### 前序遍历（Pre-Order）

根 → 左 → 右，可用于复制树结构。

```
前序遍历：8 → 3 → 1 → 6 → 10 → 14
```

#### 后序遍历（Post-Order）

左 → 右 → 根，可用于安全删除树。

```
后序遍历：1 → 6 → 3 → 14 → 10 → 8
```

#### 层序遍历（Level-Order）

按层从左到右访问，使用 BFS 实现。

```
层序遍历：8 → 3, 10 → 1, 6, 14
```

## 4. 应用场景

### 4.1 典型应用

- **动态集合操作**：支持快速查找、插入、删除
- **有序数据维护**：中序遍历即可获得有序序列
- **范围查询**：查找某范围内的所有元素
- **前驱/后继查询**：查找比某值大/小的最近元素

## 5. 与其他数据结构对比

### 5.1 BST vs AVL 树 vs 红黑树

| 特性 | BST | AVL 树 | 红黑树 |
|------|-----|--------|--------|
| **平衡性** | 不保证 | 严格平衡（高度差 ≤ 1） | 近似平衡 |
| **查找** | 平均 O(log n)，最差 O(n) | O(log n) | O(log n) |
| **插入/删除** | 平均 O(log n)，最差 O(n) | O(log n) | O(log n) |
| **旋转操作** | 无 | 可能需要多次旋转 | 最多 3 次旋转 |
| **实现复杂度** | 简单 | 较复杂 | 复杂 |
| **适用场景** | 数据量小或近乎有序 | 查找密集型 | 插入/删除密集型 |

### 5.2 BST vs 哈希表

| 特性 | BST | 哈希表 |
|------|-----|--------|
| **有序性** | 有序 | 无序 |
| **查找** | O(log n) | 平均 O(1) |
| **范围查询** | 高效 | 不支持 |
| **空间** | O(n) | O(n) + 哈希表开销 |
| **最差情况** | O(n)（退化） | O(n)（哈希冲突） |
