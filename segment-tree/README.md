# 线段树（Segment Tree）

## 1. 概述

### 1.1 定义

线段树是一种基于分治思想的二叉树结构，用于高效处理区间查询和区间更新问题。每个节点代表一个区间，叶子节点代表单个元素，内部节点代表其子节点区间的合并结果。

### 1.2 核心特点

- **高效查询**：区间查询时间复杂度为 O(log n)
- **灵活更新**：支持单点更新和区间更新
- **空间换时间**：使用 O(n) 额外空间换取操作效率
- **通用性强**：可处理求和、最值、GCD 等多种区间聚合操作

### 1.3 时间复杂度

| 操作 | 时间复杂度 |
|------|-----------|
| 建树 | O(n) |
| 单点更新 | O(log n) |
| 区间查询 | O(log n) |
| 区间更新（懒标记） | O(log n) |

## 2. 数据结构原理

### 2.1 树形结构

线段树是一棵完全二叉树，对于长度为 n 的数组，线段树的结构如下：

```
            [0, 7]              <- 根节点，代表整个区间
           /      \
      [0, 3]      [4, 7]        <- 内部节点
      /    \      /    \
   [0,1]  [2,3] [4,5]  [6,7]
   /  \   /  \   /  \   /  \
 [0] [1] [2] [3] [4] [5] [6] [7]  <- 叶子节点
```

### 2.2 节点表示

每个节点存储以下信息：
- 区间范围 [left, right]
- 区间聚合值（如区间和、区间最大值等）
- 懒标记（用于区间更新优化）

### 2.3 数组存储

使用数组存储线段树，根节点从下标 1 开始（1-indexed），对于节点 i：
- 左子节点：`2 * i`
- 右子节点：`2 * i + 1`
- 父节点：`i / 2`

若根节点从下标 0 开始（0-indexed），则公式变为：
- 左子节点：`2 * i + 1`
- 右子节点：`2 * i + 2`
- 父节点：`(i - 1) / 2`

1-indexed 更常用，因为父子关系的公式更简洁，且下标 0 空置不影响实际使用。

对于长度为 n 的数组，通常需要 **4n** 大小的数组来存储线段树。

**为什么是 4n？**

线段树是一棵完全二叉树，叶子节点数量为 n。将 n 向上取整到最近的 2 的幂次 2^⌈log₂n⌉，则满二叉树的总节点数为 2 × 2^⌈log₂n⌉ - 1。由于 2^⌈log₂n⌉ < 2n，因此总节点数 < 4n。使用 4n 作为数组大小可以保证不会越界，且无需精确计算。

## 3. 核心操作详解

### 3.1 建树（Build）

#### 算法思路

递归地将区间一分为二，直到区间长度为 1（叶子节点），然后回溯计算父节点的聚合值。

#### 图解过程

以数组 `[1, 3, 5, 7, 9, 11]` 建立区间和线段树为例：

```
原数组: [1, 3, 5, 7, 9, 11]

建树过程：
1. 根节点 [0, 5] = 1+3+5+7+9+11 = 36
2. 左子树 [0, 2] = 1+3+5 = 9
   - 左子节点 [0, 1] = 1+3 = 4
     - 叶子 [0] = 1
     - 叶子 [1] = 3
   - 右子节点 [2, 2] = 5
3. 右子树 [3, 5] = 7+9+11 = 27
   - 左子节点 [3, 4] = 7+9 = 16
     - 叶子 [3] = 7
     - 叶子 [4] = 9
   - 右子节点 [5, 5] = 11

最终线段树：
                 [0,5]=36
                /        \
         [0,2]=9        [3,5]=27
         /      \        /      \
     [0,1]=4  [2,2]=5 [3,4]=16 [5,5]=11
     /    \           /    \
  [0]=1 [1]=3     [3]=7 [4]=9
```

---

### 3.2 区间查询（Query）

#### 算法思路

在树上递归查找，将查询区间分解为若干个不相交的子区间，合并这些子区间的结果得到最终答案。

#### 区间分解过程

查询区间 [1, 4] 的和：

```
查询 [1, 4]：
1. 根节点 [0, 5]：与 [1, 4] 有交集，继续向下
2. 左子树 [0, 2]：与 [1, 4] 有交集
   - [0, 1]：与 [1, 4] 有交集
     - [0]：不在 [1, 4] 内，返回 0
     - [1]：在 [1, 4] 内，返回 3
   - [2, 2]：在 [1, 4] 内，返回 5
3. 右子树 [3, 5]：与 [1, 4] 有交集
   - [3, 4]：完全包含在 [1, 4] 内，直接返回 16
   - [5, 5]：不在 [1, 4] 内，返回 0

最终结果：3 + 5 + 16 = 24
```

#### 图解说明

```
查询 [1, 4] 的分解过程：

                 [0,5]
                /      \
         [0,2]          [3,5]      <- [3,4] 完全包含，直接返回
         /    \          /    \
     [0,1]  [2,2]   [3,4]  [5,5]
     /  \            /  \
  [0]  [1]        [3]  [4]
   ↓     ✓         ✓    ✓
  跳过  返回3    返回7  返回9

结果 = 3 + 5 + (7+9) = 24
```

### 3.3 单点更新（Update）

#### 算法思路

从根节点出发，沿着路径找到目标叶子节点，更新叶子节点的值，然后回溯更新路径上所有祖先节点的值。

#### 路径更新过程

将索引 2 的值从 5 更新为 10：

```
更新 arr[2] = 10：

路径上的节点更新：
1. [2, 2]：5 -> 10
2. [0, 2]：9 -> 14 (1+3+10)
3. [0, 5]：36 -> 41 (14+27)

更新前后对比：
更新前：                 更新后：
     [0,5]=36                 [0,5]=41
    /        \               /        \
[0,2]=9   [3,5]=27      [0,2]=14   [3,5]=27
```

## 4. 懒标记机制

### 4.1 问题背景

普通的区间更新需要遍历区间内的所有叶子节点，时间复杂度为 O(n)，效率低下。

**懒标记（Lazy Propagation）** 的核心思想：延迟更新，只在需要时才将更新操作下推到子节点。

### 4.2 懒标记原理

当对一个区间进行更新时：
1. 如果当前区间完全包含在更新区间内，直接更新当前节点的值
2. 将更新操作记录在当前节点的懒标记中，不立即下推
3. 当后续查询或更新涉及到子节点时，再将懒标记下推

### 4.3 下推操作（PushDown）

#### 图解说明

给区间 [0, 3] 的所有元素加 2：

```
初始状态：
         [0,3]=10
        /        \
    [0,1]=4     [2,3]=6
    /    \       /    \
  [0]=1 [1]=3 [2]=5 [3]=1

执行 update(0, 3, +2)：
1. 根节点 [0,3] 完全包含在 [0,3] 内
2. 更新 tree[1] = 10 + 2*4 = 18
3. 设置 lazy[1] = 2（懒标记）

状态：
         [0,3]=18 (lazy=2)
        /        \
    [0,1]=4     [2,3]=6
    /    \       /    \
  [0]=1 [1]=3 [2]=5 [3]=1

查询 [0, 1] 时，需要下推懒标记：
1. 父节点 lazy[1] = 2，需要下推
2. 左子节点 [0,1]：tree += lazy * 区间长度 = 4 + 2*2 = 8
3. 右子节点 [2,3]：tree += lazy * 区间长度 = 6 + 2*2 = 10
4. 子节点 lazy += 父节点 lazy
5. 父节点 lazy 清零

下推后：
         [0,3]=18 (lazy=0)
        /        \
    [0,1]=8     [2,3]=10
   (lazy=2)    (lazy=2)
```

## 6. 与其他数据结构对比

### 6.1 线段树 vs 树状数组（Fenwick Tree）

| 特性 | 线段树 | 树状数组 |
|------|--------|----------|
| **功能** | 支持区间更新、区间查询 | 主要支持单点更新、区间查询 |
| **实现复杂度** | 较复杂 | 简单 |
| **空间复杂度** | O(4n) | O(n) |
| **常数因子** | 较大 | 较小 |
| **通用性** | 强（支持各种聚合操作） | 较弱（主要支持求和） |
| **区间更新** | 原生支持（懒标记） | 需要差分技巧 |

### 6.2 适用场景分析

**选择线段树的情况**：
- 需要区间更新功能
- 需要支持多种聚合操作（最值、GCD 等）
- 区间查询的区间长度不固定

**选择树状数组的情况**：
- 只需要单点更新、区间求和
- 对常数因子敏感，追求极致性能
- 代码实现需要简洁

### 6.3 复杂度对比

| 操作 | 线段树 | 树状数组 |
|------|--------|----------|
| 建树 | O(n) | O(n log n) |
| 单点更新 | O(log n) | O(log n) |
| 区间查询 | O(log n) | O(log n) |
| 区间更新 | O(log n) | O(log n)（差分） |

## 7. 实现细节与优化

### 7.1 常见实现技巧

#### 递归实现 vs 迭代实现

**递归实现**：
- 代码清晰易懂
- 函数调用开销较大
- 适合学习和理解

**迭代实现**：
- 性能更优
- 代码较复杂
- 适合生产环境

#### 数组大小计算

对于长度为 n 的数组，线段树数组大小：
- 理论最小：2 * 2^⌈log₂n⌉ - 1
- 实践常用：4 * n

### 7.2 空间优化

#### 动态开点

当区间范围很大但实际使用的点很少时，可以动态开点。

#### 离散化

将大范围的坐标映射到小范围。

### 7.3 边界处理

#### 区间表示

- **闭区间 [l, r]**：常用，易于理解
- **左闭右开 [l, r)**：某些场景更方便

#### 溢出处理

计算中点时防溢出：使用 `start + (end - start) / 2` 代替 `(start + end) / 2`。
